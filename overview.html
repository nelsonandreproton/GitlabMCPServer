GitlabMCPServer

HTTP gateway exposing GitLab MCP tools over a URL — built for Notion's URL-only MCP client

Node.js @zereight/mcp-gitlab Docker + Caddy Deployed (Hetzner + sslip)

1. What this project does

Purpose

A thin Node.js proxy that lets URL-only MCP clients (Notion) reach a GitLab MCP server. Notion can only supply a URL — no custom headers — so the gateway reads the user's GitLab Personal Access Token from ?token= in the query string and forwards it as an Authorization: Bearer header to the upstream @zereight/mcp-gitlab running in remote-auth mode. Each colleague connects with their own PAT and acts as themselves in GitLab.

The gateway turns ?token=PAT into Authorization: Bearer PAT. Notion auth must be set to None — the header-based "API key" option is ignored.

3. Anonymous usage tracking

Counts distinct users per day without storing identity. Each token is SHA-256 hashed (one-way — raw tokens never touch disk) and deduplicated per UTC day. Read counts at GET /stats; only counts are exposed, never the hashes. State persists on a Docker named volume so it survives redeploys.

EndpointGET /stats → {"2026-06-01": 5} StorageSTATS_FILE on named volume PrivacySHA-256(token), no raw token stored
