mcp-gitlab-gateway Azure VM

Step-by-step deployment guide — same VM as the self-hosted GitLab instance

Notion / Claude → HTTPS :443 → nginx (TLS termination)
                                    ↓
                                    server.js :8808 (gateway — injects token)
                                    ↓
                                    @zereight/mcp-gitlab :3002 (MCP upstream)
                                    ↓
                                    GitLab API (localhost, same VM)

Steps

  1. SSH into the VM & install Node.js
  2. Clone the repository
  3. Create the environment file
  4. Install the MCP package
  5. Create systemd services
  6. Configure nginx reverse proxy
  7. Open the Azure firewall port
  8. Verify end-to-end
1
SSH into the VM & install Node.js

Connect to the Azure VM. The GitLab instance is already running here.

ssh hetzner

Check if Node.js is already installed:

node --version   # need v18 or newer
npm --version

If Node.js is missing or older than v18, install via NodeSource:

curl -fsSL https://deb.nodesource.com/setup_22.x | sudo bash -
sudo apt-get install -y nodejs

Verify:

node --version   # should print v22.x.x
2
Clone the repository

Clone into /opt/mcp-gitlab-gateway (or any path you prefer).

sudo git clone https://github.com/YOUR_ORG/GitlabMCPServer.git /opt/mcp-gitlab-gateway
sudo chown -R $USER:$USER /opt/mcp-gitlab-gateway
cd /opt/mcp-gitlab-gateway
If the repo is on your self-hosted GitLab, replace the URL with your internal GitLab URL. Make sure the VM can reach it — it's on the same VNet so it should work directly.

If you haven't pushed to GitLab yet, you can also copy the files manually:

# From your Windows machine (run in PowerShell):
scp C:\dev\GitlabMCPServer\server.js hetzner:/opt/mcp-gitlab-gateway/
scp C:\dev\GitlabMCPServer\test-body.json hetzner:/opt/mcp-gitlab-gateway/
3
Create the environment file

Create /opt/mcp-gitlab-gateway/.env — this is used by both systemd services.

sudo nano /opt/mcp-gitlab-gateway/.env
/opt/mcp-gitlab-gateway/.env
# GitLab API — internal URL (same VNet, no need to go through nginx)
GITLAB_API_URL=https://np-gitlab.swedencentral.cloudapp.azure.com/api/v4

# Required to bypass self-signed cert on GitLab
NODE_TLS_REJECT_UNAUTHORIZED=0

# Placeholder — actual token comes per-request via Authorization header
GITLAB_PERSONAL_ACCESS_TOKEN=placeholder

# Upstream config (mcp-gitlab process)
STREAMABLE_HTTP=true
REMOTE_AUTHORIZATION=true
HOST=127.0.0.1
PORT=3002

# Gateway config (server.js)
GATEWAY_PORT=8808
UPSTREAM_PORT=3002

Lock down permissions — this file contains sensitive config:

chmod 600 /opt/mcp-gitlab-gateway/.env
4
Install the MCP package

Pre-install @zereight/mcp-gitlab so it doesn't need to be fetched at runtime. This avoids the cold-start delay and works offline.

cd /opt/mcp-gitlab-gateway
npm install @zereight/mcp-gitlab

Then update /etc/systemd/system/mcp-gitlab-upstream.service (next step) to run the local binary instead of npx.

5
Create systemd services

Two services: one for the MCP upstream, one for the gateway. They must start in order.

Service 1 — MCP upstream

sudo nano /etc/systemd/system/mcp-gitlab-upstream.service
/etc/systemd/system/mcp-gitlab-upstream.service
[Unit]
Description=GitLab MCP Upstream (streamable-http)
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/mcp-gitlab-gateway
EnvironmentFile=/opt/mcp-gitlab-gateway/.env
ExecStart=/usr/bin/node /opt/mcp-gitlab-gateway/node_modules/.bin/mcp-gitlab
Restart=always
RestartSec=5
StandardOutput=journal
StandardError=journal
SyslogIdentifier=mcp-gitlab-upstream

[Install]
WantedBy=multi-user.target

Service 2 — Gateway proxy

sudo nano /etc/systemd/system/mcp-gitlab-gateway.service
/etc/systemd/system/mcp-gitlab-gateway.service
[Unit]
Description=GitLab MCP Gateway (HTTP proxy)
After=network.target mcp-gitlab-upstream.service
Requires=mcp-gitlab-upstream.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/mcp-gitlab-gateway
Environment=PORT=8808
Environment=UPSTREAM_PORT=3002
EnvironmentFile=/opt/mcp-gitlab-gateway/.env
ExecStart=/usr/bin/node /opt/mcp-gitlab-gateway/server.js
Restart=always
RestartSec=5
StandardOutput=journal
StandardError=journal
SyslogIdentifier=mcp-gitlab-gateway

[Install]
WantedBy=multi-user.target

Enable and start both services:

sudo systemctl daemon-reload
sudo systemctl enable mcp-gitlab-upstream mcp-gitlab-gateway
sudo systemctl start mcp-gitlab-upstream

# Wait a couple of seconds for upstream to be ready, then start gateway
sleep 3
sudo systemctl start mcp-gitlab-gateway

Check both are running:

sudo systemctl status mcp-gitlab-upstream
sudo systemctl status mcp-gitlab-gateway

# Follow live logs
sudo journalctl -u mcp-gitlab-upstream -f
sudo journalctl -u mcp-gitlab-gateway -f
Upstream log should show: GitLab MCP Server running with Streamable HTTP transport and Endpoint: http://127.0.0.1:3002/mcp
6
Configure nginx reverse proxy

Add a new nginx site. Replace mcp-gitlab.np-gitlab.swedencentral.cloudapp.azure.com with whatever hostname or IP you want to use.

sudo nano /etc/nginx/sites-available/mcp-gitlab-gateway
/etc/nginx/sites-available/mcp-gitlab-gateway
server {
    listen 443 ssl;
    server_name mcp-gitlab.np-gitlab.swedencentral.cloudapp.azure.com;

    # Use the same cert as GitLab, or get a new one with certbot
    ssl_certificate     /etc/ssl/certs/gitlab.crt;
    ssl_certificate_key /etc/ssl/private/gitlab.key;

    location /mcp {
        proxy_pass         http://127.0.0.1:8808;
        proxy_http_version 1.1;

        # Required for SSE / streaming responses
        proxy_buffering    off;
        proxy_cache        off;
        proxy_read_timeout 86400s;
        proxy_send_timeout 86400s;

        # Tell nginx not to buffer even if misconfigured
        add_header X-Accel-Buffering no;

        proxy_set_header Host              $host;
        proxy_set_header X-Real-IP         $remote_addr;
        proxy_set_header X-Forwarded-For   $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /health {
        proxy_pass http://127.0.0.1:8808;
    }
}

# Redirect HTTP to HTTPS
server {
    listen 80;
    server_name mcp-gitlab.np-gitlab.swedencentral.cloudapp.azure.com;
    return 301 https://$host$request_uri;
}
If you don't have a named subdomain, you can skip the server_name line and use the VM's public IP directly. In that case, use a self-signed cert or skip TLS for internal-only use.

Enable the site and reload nginx:

sudo ln -s /etc/nginx/sites-available/mcp-gitlab-gateway /etc/nginx/sites-enabled/
sudo nginx -t        # test config — must say "syntax is ok"
sudo systemctl reload nginx
7
Open the Azure firewall port

If port 443 is not already open on the VM's Network Security Group (it should be if GitLab is already HTTPS), no action is needed. Verify in the Azure Portal:

  • Go to Azure Portal → Virtual Machines → your VM → Networking
  • Check Inbound port rules
  • Confirm there is a rule allowing TCP 443 from any source (or from your specific IPs)
Port 8808 does not need to be open externally — nginx handles TLS on 443 and proxies internally to 8808.

If port 443 is missing, add it:

# Using Azure CLI (if installed)
az network nsg rule create \
  --resource-group YOUR_RESOURCE_GROUP \
  --nsg-name YOUR_NSG_NAME \
  --name AllowHTTPS \
  --priority 300 \
  --protocol Tcp \
  --destination-port-range 443 \
  --access Allow
8
Verify end-to-end

From your local machine, test the full stack:

1. Health check

curl https://mcp-gitlab.np-gitlab.swedencentral.cloudapp.azure.com/health

Expected: {"status":"ok","upstream":"127.0.0.1:3002"}

2. MCP initialize handshake

curl -s -X POST \
  "https://mcp-gitlab.np-gitlab.swedencentral.cloudapp.azure.com/mcp?token=YOUR_GITLAB_PAT" \
  -H "Content-Type: application/json" \
  -H "Accept: application/json, text/event-stream" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2024-11-05","capabilities":{},"clientInfo":{"name":"test","version":"1.0"}}}'

Expected response:

event: message
data: {"result":{"protocolVersion":"2024-11-05","capabilities":{"tools":{}},"serverInfo":{"name":"better-gitlab-mcp-server","version":"2.1.16"}},"jsonrpc":"2.0","id":1}

3. Connect from Notion

In Notion → Settings → Connections → Add MCP Server, use:

https://mcp-gitlab.np-gitlab.swedencentral.cloudapp.azure.com/mcp?token=USER_GITLAB_PAT

Each user enters their own GitLab Personal Access Token in the URL. Tokens need at least read_api scope; api scope for write operations.

If the health check passes and the MCP handshake returns a valid response, the deployment is complete.

Useful commands for ongoing operations

# Restart after code changes
sudo systemctl restart mcp-gitlab-upstream mcp-gitlab-gateway

# Pull latest code and restart
cd /opt/mcp-gitlab-gateway
sudo git pull
sudo systemctl restart mcp-gitlab-upstream mcp-gitlab-gateway

# View logs
sudo journalctl -u mcp-gitlab-gateway -n 50
sudo journalctl -u mcp-gitlab-upstream -n 50

# Check both services at once
sudo systemctl status mcp-gitlab-upstream mcp-gitlab-gateway
